# V2.3.1 — Controle de Execução

Data: 2026-07-15

## Problema

Os comandos de pausa, retomada e cancelamento não apresentavam comportamento confiável durante uma execução ativa.

## Correção

- eventos de sincronização separados para pausa e cancelamento;
- botão Pausar muda visualmente para Retomar;
- status visível PAUSADO, EM EXECUÇÃO e CANCELADO;
- cancelamento interrompe a thread antes de qualquer clique pendente;
- sinais ainda agendados ou em execução são marcados como CANCELADO;
- histórico registra pausa, retomada e cancelamento;
- controles ficam desabilitados quando não existe execução ativa.
